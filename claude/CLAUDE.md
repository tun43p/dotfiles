## Git Conventions

If you need to commit something, you must always follow these rules and never deviate from them:

- All commits must be signed; if they aren't, there's a problem.
- The `init: initial commit` must **ALWAYS** be the first commit for new projects, and it must ONLY and ALWAYS consist of a README.md and a LICENSE file.
- All commits must **ALWAYS** be conventional commits and as detailed as possible, be atomic and self-contained.
- NEVER add `Co-authored-by: Claude <claude@anthropic.com>` to any commit or other metadata.
